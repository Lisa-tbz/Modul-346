## Teil A
### Reverse Proxy
Ein Reverse Proxy ist ein Server, der zwischen dem Client (z. B. Browser) internen Services steht.
Er nimmt Anfragen aus dem Internet entgegen und leitet sie intern an den passenden Service weiter (bspw. basierend auf Pfaden).
Dadurch können Lasten verteilt werden (Load Balancing) und Sicherheit, Performance sowie Flexibilität der gesamten Infrastruktur werden verbessert.

### Welche Teile der Cloud-Init datei machen keinen Sinn?
Hardcoded values für den Username und PW sind untypisch und sicherheitsmässig fragwürdig.
Idempotenz bei DB-dump nicht vorhanden (man könnte Flag setzen, um Zustand vor Import zu checken).

# Teil B - Vertikale Skalierung
### Notwendige Schritte
#### Disk Storage
Instanz muss nicht heruntergefahren sein, geht im laufenden Betrieb.
Speicher -> Volumes -> Aktionen -> Volume ändern

#### Instance Type 
Instanz muss heruntergefahren sein (Fehlermeldung und Option ausgegraut), geht nicht im laufenden Betrieb.
Instanz -> Aktionen -> Instance-Einstellungen -> Instance-Typ ändern

### DNS
Das DNS (Domain Name System) agiert as Telefonbuch, 
welches Domainnamen (wie something.com) in IP-Adressen übersetzt,
was eine Kommunikation ermöglicht. 
Somit kann man eine Website basierend auf der Domain aufrufen und muss
sich die IP-Adresse nicht merken.
